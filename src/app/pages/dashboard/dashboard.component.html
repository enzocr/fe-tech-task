Dashboard

  • Debe utilizar la estructura Angular 17 provista por el profesor en las clases técnicas
  • Debe consumir por medio de HTTP los métodos del API generado en la tarea 1 técnica CRUD back-end
  • Deben existir dos tipos de roles para usuarios: SUPER-ADMIN-ROLE y USER (provenientes de la tarea 1 CRUD back-end)
  • Debe crear una página para administrar los productos y otra para administrar las categorías donde se listen todos los datos de las entidades
  • Debe crear un componente para registrar nuevos productos y categorías, también debe reutilizar ese componente para actualizar ambas entidades
  • Debe existir un inicio de sesión y un registro de usuarios
  • Deben registrarse los datos en una base de datos MySQL o MariaDB de forma local, no cloud.
  • Debe crear la funcionalidad para eliminar productos y categorías
  • La consulta de productos y categorías será permitida para todos los usuarios si están autenticados
  • El registro, actualización y borrado de productos y categorías únicamente podrá realizarlo el usuario con rol SUPER-ADMIN-ROLE
  • Debe existir una página y un componente para mostrar los usuarios registrados en el sistema, solo el SUPER-ADMIN-ROLE podrá ingresar a ese componente.
